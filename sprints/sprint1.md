## Retrospectiva
**Objetivo general**: Desarrollar el núcleo funcional inicial del sistema.

### ¿Cómo nos fue con la entrega del software funcional y terminado?
- Se logró entregar una base de software funcional y bien enfocada para mejoras en el futuro.
- Con respecto a la planificación, no se logró entregar un implementación "Terminada". Ninguna User Story alcanzó el 100% de finalización.
- Tasks quedaron en "In progress" al finalizar el sprint, impidiendo el cierre de las User Story correspondientes.

### ¿Qué nos causó problemas?
- **Planificación:** La falta de organización y la poca importancia a los tiempos provocaron que el equipo no lograra completar todas las User Story ni las task asignadas a este sprint.
- **Seguimiento:** El sprint planning muestra tasks en "In progress". Esto sugiere que no se logró su finalización a tiempo, aunque esto va de la mano con **Planificación**.

### ¿Qué vamos a mejorar?
- **Planificación:** Se hará más enfasis en el refinamiento de las HUs antes del Sprint 2, para asegurar que las tasks sean mas claras y mejor definidas.
- **Organización:** Se priorizara con otro enfoque el trabajo del Sprint 2. El objetivo será finalizar el trabajo pendiente del Sprint 1 antes de dar inicio al Sprint 2.
- **Tiempos:** Se reevaluará desde cero el estimado de las tareas restantes. Se descarta el Sprint 1 como parámetro/guia para la "velocidad" y se realizará una estimación acorde al tiempo que nos proponen.
- **Enfoque:** El objetivo del equipo debe ser analizar con mas cuidado el Sprint Backlog, de modo que luego no haya un frenado en la realizacion de las implementaciones propuestas.
